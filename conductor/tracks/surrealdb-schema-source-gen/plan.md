# SurrealDB Schema Source Generator ‚ÄĒ Plan

Single owner, ~1 week. New package `Oasis.SurrealDb.SourceGen` plus
typed-builder companion `SurrealQuery<T>` in the existing `Client`
package. No multi-wave structure ‚ÄĒ small enough for one track.

## Phase 1 ‚ÄĒ Bootstrap

1. [ ] Create `packages/Oasis.SurrealDb.SourceGen/` with
       `Oasis.SurrealDb.SourceGen.csproj` (`netstandard2.0` only ‚ÄĒ source
       generators must); version 0.1.0; `IsRoslynComponent=true`;
       package metadata mirrors the other three packages.
2. [ ] Create `tests/Oasis.SurrealDb.SourceGen.Tests/` (xunit +
       FluentAssertions + `Microsoft.CodeAnalysis.CSharp.SourceGenerators.Testing.XUnit`,
       same version as existing analyzer tests).
3. [ ] Wire both projects into `oasis-sleek.sln`.
4. [ ] Add `ProjectReference` from `Oasis.SurrealDb.SourceGen.csproj` to
       `Oasis.SurrealDb.Schema.csproj` (re-uses the existing Mermaid
       parser; no duplicate parser implementation).

## Phase 2 ‚ÄĒ `IIncrementalGenerator` core

5. [ ] `OasisSurrealDbSchemaGenerator : IIncrementalGenerator`. Wires:
       - `context.AdditionalTextsProvider.Where(file => file.Path.EndsWith(".mermaid"))`
       - `Select` reads + parses via `MermaidParser` from the Schema
         package, produces a `(filePath, MermaidSchemaModel)` tuple
       - `RegisterSourceOutput` emits the generated `.cs` per entity
6. [ ] `CSharpTypeMapper` ‚ÄĒ pure function `(SurrealType, IsOption) ‚Üí
       CSharpTypeRef`. Deterministic table: `string‚Üístring`, `int‚Üílong`,
       `decimal‚Üídecimal`, `datetime‚ÜíDateTimeOffset`,
       `duration‚ÜíTimeSpan`, `bool‚Üíbool`, `record<T>‚ÜíRecordId<T>`,
       `array<X>‚ÜíIReadOnlyList<X>`, `option<X>‚Üínullable wrapper`. Unknown
       types throw `NotSupportedException` with the offending Mermaid
       type in the message (forces explicit support before silently
       generating wrong code).
7. [ ] `CSharpEmitter` ‚ÄĒ given a `MermaidSchemaModel + namespace + class
       name`, produces a `partial class <Name>` source with:
       - `[JsonPropertyName("snake_case_name")]` per property
       - C# type per the mapper
       - Static `SchemaName` property returning the SurrealDB table name
       - For `string ASSERT INSIDE [...]` fields: a sibling enum
         declaration in the same generated file, plus the property typed
         as that enum
       - File header comment `// <auto-generated>` so the compiler
         skips formatting/style diagnostics
       - Deterministic ordering: entities alphabetical by name; properties
         in `.mermaid` source order
8. [ ] Annotation extensions for the existing Mermaid annotation DSL
       (additions in the Schema package, NOT in the source-gen package
       ‚ÄĒ keeps the schema package the only Mermaid-aware code):
       - `@surreal.csharp.property name=<C#PropertyName>` ‚ÄĒ overrides the
         default snake_case‚ÜíPascalCase translation per-field
       - `@surreal.csharp.skip` ‚ÄĒ generator skips this field (for fields
         that exist in the schema but aren't projected to the application
         layer)
       - `@surreal.csharp.namespace <ns>` ‚ÄĒ entity-level override of the
         per-assembly namespace
9. [ ] Unit tests in `tests/Oasis.SurrealDb.SourceGen.Tests/`:
       - Type-mapping table (one test per SurrealType row)
       - `Generator_emits_partial_class_per_entity` (single-entity input)
       - `Generator_emits_one_file_per_mermaid_source` (multi-entity)
       - `Generator_is_deterministic` (run twice, byte-compare output)
       - `Annotation_csharp_property_overrides_default_translation`
       - `Annotation_csharp_skip_omits_field`
       - `String_assert_inside_generates_enum`
       - `Unknown_surreal_type_throws_with_descriptive_message`

## Phase 3 ‚ÄĒ `RecordId<T>` typed companion

10. [ ] Add `packages/Oasis.SurrealDb.Client/RecordId{T}.cs` ‚ÄĒ generic
        struct wrapping `RecordId`. `T` is a phantom type parameter (no
        runtime presence beyond a marker; `T` must implement a marker
        interface `ISurrealRecord` that the source generator adds to
        every generated POCO).
11. [ ] Implicit conversion from `RecordId<T>` to `RecordId`; explicit
        conversion from `RecordId` to `RecordId<T>` with a runtime
        `SchemaName` check (throws `InvalidCastException` if the
        underlying table-string doesn't match `T.SchemaName`).
12. [ ] JSON converter inherits from existing `RecordIdJsonConverter`;
        round-trip semantics identical.
13. [ ] Unit tests in `tests/Oasis.SurrealDb.Client.Tests/RecordIdGenericTests.cs`:
        - Implicit conversion preserves data
        - Explicit conversion with matching schema succeeds
        - Explicit conversion with mismatched schema throws
        - JSON round-trip preserves table and id

## Phase 4 ‚ÄĒ `SurrealQuery<T>` typed builder

14. [ ] Add `packages/Oasis.SurrealDb.Client/Query/SurrealQuery{T}.cs`.
        Static `SurrealQuery<T>.From()` returns the typed builder.
        Reads `T.SchemaName` to know the SurrealDB table.
15. [ ] `ExpressionTranslator` ‚ÄĒ visits `Expression<Func<T, bool>>` and
        emits SurrealQL + parameter bag. Supported nodes:
        - `MemberExpression` on the lambda parameter ‚Üí column name from
          `[JsonPropertyName]`
        - `ConstantExpression` ‚Üí parameter binding (auto-named
          `$_p0, $_p1, ...`)
        - `BinaryExpression`: `==, !=, <, <=, >, >=, &&, ||`
        - `MethodCallExpression`: `string.IsNullOrEmpty(x)` ‚Üí
          `x = NONE OR x = ""`; `array.Contains(value)` ‚Üí `value INSIDE array`
        - Unsupported nodes throw `NotSupportedException` with a static
          fallback recipe: *"This predicate isn't supported by the typed
          builder. Fall back to SurrealQuery.Of(...).Where(...) for this
          query. Specific node: {kind}."*
16. [ ] `.OrderBy(Expression<Func<T, object>>)`,
        `.OrderByDescending(...)`, `.ThenBy(...)`, `.Limit(int)`,
        `.Start(int)`, `.Fetch(Expression<Func<T, object>>)`,
        `.Select<TResult>(Expression<Func<T, TResult>>)`.
17. [ ] `SurrealQuery<T>` is **convertible to `SurrealQuery`** so the
        existing executor path consumes typed queries without surface
        widening. Implicit operator + explicit `.AsUntyped()` method.
18. [ ] Unit tests in `tests/Oasis.SurrealDb.Client.Tests/Query/SurrealQueryTypedTests.cs`:
        - `From_emits_SELECT_star_FROM_table_per_SchemaName`
        - `Where_equality_emits_correct_SurrealQL_and_parameter`
        - `Where_compound_AND_chains_AND` (regression for the M3 clause
          tracking fix)
        - `OrderBy_chained_with_ThenBy_emits_correct_clauses`
        - `Select_projection_emits_field_list`
        - `Unsupported_method_call_throws_with_fallback_recipe`
        - Byte-identical to untyped equivalent (golden-file)
